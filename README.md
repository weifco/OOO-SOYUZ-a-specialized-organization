# Информационная система ООО «СОЮЗ»

Учебный прототип разработан для курсовой работы по теме: **«Оптимизация деятельности строительной фирмы ООО СОЮЗ на основе разработки информационной системы»**.

## Состав проекта

- `app/server.py` — основной сервер приложения без внешних фреймворков;
- `app/database.py` — создание базы данных и заполнение тестовыми данными;
- `app/construction_firm.db` — готовая SQLite-база с тестовыми данными;
- `app/static/style.css` — стили интерфейса;
- `docs/schema.sql` — структура базы данных;
- `docs/technical_specification.md` — техническое задание;
- `docs/user_manual.md` — руководство пользователя;
- `docs/testing_report.md` — отчет о тестировании;
- `tests/test_scenarios.md` — тестовые сценарии.

## Запуск

1. Открыть корневую папку проекта, куда распакован архив.
2. Выполнить команду:

```bash
python server.py
```

Если запуск выполняется из папки `app`, команда также будет работать:

```bash
cd app
python server.py
```

3. Перейти в браузере по адресу:

```text
http://127.0.0.1:8000[README.md](README.md)
```

Для пересоздания базы данных с тестовыми данными из корневой папки:

```bash
python server.py --reset
```

## Добавленные функции

1. **Разграничение прав пользователей**: меню и операции зависят от роли пользователя.
2. **Мои задачи**: исполнитель видит назначенные ему задачи и может обновлять готовность.
3. **Заявки на ресурсы**: менеджер или инженер создают заявку, руководитель или администратор согласует либо отклоняет.
4. **Контроль минимальных остатков**: ресурсы с низким доступным остатком подсвечиваются.
5. **Реестр проектных рисков**: фиксируются риски по объектам, вероятность, влияние и меры реагирования.
6. **Оперативные уведомления**: на панели отображаются просрочки, низкие остатки, заявки и риски.
7. **Экспорт отчета в CSV**: руководитель, администратор и бухгалтер могут выгрузить отчет по объектам.
8. **Журнал действий**: фиксируются значимые операции пользователей.

## Назначение проекта

Система демонстрирует автоматизацию учета строительных объектов, задач, ресурсов, договоров и управленческих решений. Прототип может использоваться в курсовой работе как практическая часть, подтверждающая проектирование структуры базы данных, разработку основных модулей, пользовательского интерфейса, разграничения доступа и тестирование.
